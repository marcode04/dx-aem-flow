# LinkedIn Strategy — Personal Brand as a Developer

## Why LinkedIn Is Your #1 Channel

- Personal profiles get **8x more engagement** than company pages — and you ARE the brand
- Recruiters, engineering managers, and fellow devs live here
- Your open-source project IS your portfolio — LinkedIn lets you narrate it
- Every post builds your reputation as "the developer who built that AI workflow toolkit"
- Zero cost. No ads needed. Organic personal posts win.

## Personal Brand Strategy

### Profile Optimization

**Headline formula:**
> "Building the future of AI-powered development | Creator of dx-aem-flow (73+ AI skills for enterprise dev teams) | Open Source"

**About section** — Tell the story:
- Why you built this (pain point you experienced)
- What it does (one paragraph, quantified)
- What makes it different (platform-agnostic, autonomous agents)
- CTA: Link to GitHub repo + docs site

**Featured section:**
- Pin your best-performing post
- Link to the docs site
- Link to GitHub repo
- Link to a demo video (when available)

**Banner image:**
- Create a branded banner showing: "73 AI Skills | 4 Plugins | Every AI Platform"
- Use the sprint lifecycle visual from the docs site

### LinkedIn Post Templates

**Template 1: The Structure Gap**
```
Every dev team uses AI now. Copilot, Cursor, Claude Code — pick your tool.

But here's what I noticed on enterprise projects:

→ Developer A prompts AI, gets decent output
→ Developer B prompts the same AI, gets inconsistent output
→ No one validates requirements before coding
→ No one checks if the PR meets Definition of Done
→ No audit trail of what AI analyzed or decided

The problem isn't AI intelligence. It's structure.

I built 73 structured skills that produce consistent,
enterprise-quality output regardless of who runs them.

Named artifacts at every step. Config-driven. Auditable.
Open source: [link]

What's the biggest gap you see in AI dev tools?
```

**Template 2: Commoditized vs Structured**
```
AI code generation is commoditized. Every tool does it now.

Here's what's NOT commoditized:

1. Structured requirements analysis with named artifacts
2. Plan verification before a single line of code is written
3. 6-phase quality gates (compile, lint, test, secrets, architecture, AI review)
4. Domain-specific verification (AEM dialogs, JCR content, editorial guides)
5. Autonomous agents enforcing standards in CI/CD — 24/7

That's the gap between "AI suggests code" and
"AI delivers enterprise-grade structured output."

I built an open-source plugin system that bridges this gap.
4 plugins. Consistent results every time.

Agree or disagree? 👇
```

**Template 3: Carousel Outline — "From Ticket to Verified PR: Structured AI Workflow"**
```
Slide 1: "AI code generation is commoditized. Structured enterprise output is not." (hook)
Slide 2: Step 1 — /dx-req: Fetch ticket, validate DoR, research codebase
Slide 3: Step 2 — /dx-plan: Generate implementation plan with risk analysis
Slide 4: Step 3 — /dx-step-all: Execute all steps (implement → test → review loop)
Slide 5: Step 4 — /dx-step-verify: 6-phase verification gate
Slide 6: Step 5 — /dx-pr: Create PR with full context
Slide 7: Step 6 — Autonomous PR review by AI agents
Slide 8: "All open source. All config-driven. Works on Claude Code, Copilot CLI, VS Code Chat."
Slide 9: CTA — "Star us on GitHub: [link]"
```

## Engagement Strategy (This Is the Real Growth Engine)

### Daily Routine (15-20 min)
1. Comment on 5-10 posts from target audience (engineering leaders, DevRel, AI dev tool creators)
2. Reply to all comments on your posts within 2 hours
3. Send 3-5 connection requests with personalized notes
4. Share/repost 1 relevant industry post with your take

### People to Engage With
- AI coding tool creators (Cursor, Continue, Cody teams)
- Claude Code / Anthropic DevRel
- GitHub Copilot team members
- Azure DevOps community leaders
- AEM / Adobe community managers
- DevOps influencers and thought leaders
- Engineering managers at enterprise companies
