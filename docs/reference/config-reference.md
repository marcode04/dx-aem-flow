# Config Reference: `.ai/config.yaml`

Generated by `/dx-init`, extended by `/aem-init`. All skills read this file at runtime.

## Full Schema

```yaml
# ─── Core (required) ───────────────────────────────────

project:
  name: "My Project"              # Human-readable project name
  prefix: "myproject"             # Short prefix used in naming conventions

scm:
  provider: ado                   # ado | github
  org: "https://myorg.visualstudio.com/"  # Organization URL
  project: "My ADO Project"      # ADO project name (or GitHub org)

tracker:
  provider: ado                   # ado | jira (preferred over scm.provider for work items)
  repo-id: "uuid-here"           # Repository ID (auto-discovered or manual)
  base-branch: develop            # Default PR target branch
  wiki-id: "uuid-here"           # ADO wiki UUID (for dx-doc-gen wiki posting)
  wiki-project: "My Wiki Project" # Project owning the wiki (defaults to scm.project)
  wiki-doc-root: "/Tech Docs/Sprint-wise" # Root wiki path for sprint subfolders
  wiki-pr-review-root: "/My-Wiki/PR-Review-Reports"  # Root wiki path for PR review reports
  wiki-dor-url: "https://myorg.visualstudio.com/MyProject/_wiki/wikis/MyWiki/12345/My-DoR-Page"
  wiki-dod-url: "https://myorg.visualstudio.com/MyProject/_wiki/wikis/MyWiki/12346/My-DoD-Page"

build:
  command: "mvn clean install"    # Full build command
  test: "mvn test"                # Test command
  lint-js: "npm run lint:js"      # JS lint (optional)
  lint-css: "npm run lint:css"    # CSS lint (optional)

# ─── Optional sections ─────────────────────────────────

# Multi-market / multi-brand support
markets:
  - brand: mybrand
    countries: [us, gb, de]
  - brand: otherbrand
    countries: [ca, fr]

# Cross-repo awareness — sibling repos in your project ecosystem
repos:
  - name: My-Backend-Repo
    path: ../My-Backend-Repo          # Relative path to local checkout
    role: backend                      # backend | frontend | fullstack | config
    platform: Legacy                   # Legacy | DXN (AEM only, added by /aem-init)
    ado-project: "My Backend Project"  # ADO project (if different from scm.project)
    base-branch: develop               # Default branch (defaults to scm.base-branch)
    capabilities: [fe, be, sling]      # Rule templates for sync (auto-detected if omitted)
  - name: My-Frontend-Repo
    path: ../My-Frontend-Repo
    role: frontend

# Auto-commit and auto-PR preferences
preferences:
  auto_commit: true               # Auto-commit after each step
  auto_pr: false                  # Auto-create PR after all steps

# Override plugin default prompts
overrides:
  pr-review:
    tone: "direct, no praise"     # Reviewer tone
    severity-threshold: 80        # Minimum confidence to report (0-100)
    extra-rules: |                # Additional rules appended to defaults
      - Check for CSS custom properties over hardcoded values
      - Flag console.log left in code
  pr-answer:
    persona: "senior developer, brief responses"

# ─── Jira / Confluence (when tracker.provider: jira) ──────

# jira:
#   url: "https://myorg.atlassian.net"    # Jira Cloud instance URL
#   project-key: "MYPROJ"                 # Jira project key

# confluence:
#   url: "https://myorg.atlassian.net/wiki"  # Confluence Cloud URL
#   space-key: "MYSPACE"                     # Default Confluence space

# ─── AEM section (added by /aem-init) ──────────────────

aem:
  component-path: "/apps/myproject/components/content"  # Component definitions root
  component-prefix: "myproject"   # Naming prefix (e.g., myproject-hero)
  component-group: "My Project"   # componentGroup value in .content.xml
  resource-type-pattern: "myproject/components/content/<name>"  # Resource type template
  java-package: "com.myorg.myproject"  # Java package root (if detected)
  frontend-dir: "ui.frontend/src" # Frontend source directory
  brands: [brand1, brand2]        # Brand variants (if multi-brand)
  content-paths:                  # AEM content paths to search
    - /content/mysite/en
    - /content/mysite/de
  author-url: "http://localhost:4502"  # Local AEM author (for dialog/component editing)
  author-url-qa: "https://qa-author.example.com"  # Remote QA author (for clickable links)
  publish-url: "http://localhost:4503"  # Local AEM publisher (for user-facing verification)
  publish-url-qa: "https://qa-publish.example.com"  # Remote QA publisher (for user-facing verification)
  demo-parent-path: "/content/mysite/en/demo"  # Parent path for all AI-created demo pages (aem-verify, aem-doc-gen)
  selector: "model"               # Exporter selector (if project uses custom selector)
  current-repo: "My-Repo"         # Current repo name (for cross-repo awareness)
  platform: "legacy"              # AEM platform: legacy (custom Web Components/Gulp) or dxn (@netcentric/Webpack)
  active-brands: [brand-a]           # Active brands for this repo (scopes component/page searches)
  active-markets:                 # Active markets (scopes searches by brand + country)
    - brand: brand-a
      countries: [gb, de, fr]
  qa-basic-auth:                  # QA/Stage environment auth (read by aem-demo-capture, aem-doc-gen)
    username: "myuser"            # HTTP Basic Auth username
    password: "mypassword"        # HTTP Basic Auth password
  # repos: — REMOVED. Cross-repo config is now in the top-level repos: section.
  # /aem-init enriches the top-level repos: entries with platform and ado-project fields.

# ─── Agent roles (optional) ────────────────────────────

roles:
  re-agent:
    enabled: true
    output-format: json           # json | markdown
  qa-agent:
    enabled: true
    canModifyCode: false
    requiresAEM: true
    bugPolicy:
      onlyFor: "ui-functional"
      requiresHumanConfirmation: true
```

## Field Reference

### `project`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | Human-readable project name |
| `prefix` | string | yes | Short prefix for naming conventions |

### `scm`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `provider` | string | yes | `ado` or `github` |
| `org` | string | yes | Organization URL |
| `project` | string | yes | Project name within the org |
| `repo-id` | string | no | Repository UUID (auto-discovered if not set) |
| `base-branch` | string | yes | Default PR target (e.g., `develop`, `main`) |
| `wiki-id` | string | no | ADO wiki identifier UUID (for `/dx-doc-gen`, `/dx-pr-review-report` wiki posting) |
| `wiki-project` | string | no | ADO project that owns the wiki (if different from `scm.project`) |
| `wiki-doc-root` | string | no | Root wiki path for technical documentation (sprint subfolders created beneath) |
| `wiki-pr-review-root` | string | no | Root wiki path for PR review reports — `/dx-pr-review-report` and `/dx-pr-reviews-report` create pages beneath this path |
| `wiki-dor-url` | string | yes | Full URL to the DoR wiki page — `/dx-req-dor` fetches checklist from here |
| `wiki-dod-url` | string | yes | Full URL to the DoD wiki page — `/dx-req-dod` fetches checklist from here |

### `tracker`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `provider` | string | no | `ado` or `jira`. Preferred over `scm.provider` for work item tracking. Falls back to `scm.provider` if not set. |

### `jira`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | string | yes (if jira) | Jira Cloud instance URL (e.g., `https://myorg.atlassian.net`) |
| `project-key` | string | yes (if jira) | Jira project key (e.g., `MYPROJ`) |

### `confluence`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | string | no | Confluence Cloud URL (e.g., `https://myorg.atlassian.net/wiki`) |
| `space-key` | string | no | Default Confluence space key |

### `build`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `command` | string | yes | Full build command |
| `test` | string | yes | Test execution command |
| `lint-js` | string | no | JavaScript lint command |
| `lint-css` | string | no | CSS/SCSS lint command |

### `aem`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `component-path` | string | yes | JCR path to component definitions |
| `component-prefix` | string | yes | Component naming prefix |
| `component-group` | string | no | Default componentGroup |
| `resource-type-pattern` | string | yes | Resource type template (`<name>` is replaced) |
| `java-package` | string | no | Java package root |
| `frontend-dir` | string | no | Frontend source directory |
| `brands` | list | no | Brand names for multi-brand projects |
| `content-paths` | list | no | AEM content paths to search |
| `author-url` | string | no | Local AEM author URL — for dialog/component editing (default: `http://localhost:4502`) |
| `author-url-qa` | string | no | Remote QA author URL — for clickable links in research output |
| `publish-url` | string | no | Local AEM publisher URL — for user-facing verification (default: `http://localhost:4503`) |
| `publish-url-qa` | string | no | Remote QA publisher URL — for user-facing verification on QA environment |
| `selector` | string | no | Exporter selector for JSON endpoints |
| `current-repo` | string | no | Current repo name (for cross-repo scope detection) |
| `platform` | string | no | AEM platform: `legacy` (custom Web Components/Gulp) or `dxn` (@netcentric/Webpack) |
| `active-brands` | list | no | Active brands for this repo (scopes component and page searches) |
| `active-markets` | list | no | Active markets with brand + countries (scopes searches by market) |
| `qa-basic-auth` | object | no | QA/Stage HTTP Basic Auth credentials (`username`, `password`). Used by `aem-demo-capture` and `aem-doc-gen` for authenticated QA environments. Also install `.claude/rules/qa-basic-auth.md` via `/aem-init` for the full auth handling pattern. |
| `repos` | list | no | Cross-repo awareness — sibling repos with path, role, platform, ado-project, base-branch, capabilities |

### `overrides`

Override plugin default prompts without replacing the full file.

| Override | Fields | Affects |
|----------|--------|---------|
| `pr-review` | `tone`, `severity-threshold`, `extra-rules` | `/dx-pr-review`, `/dx-pr-reviews` |
| `pr-answer` | `persona` | `/dx-pr-answer` |

### `preferences`

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `auto_commit` | bool | false | Commit after each step without asking |
| `auto_pr` | bool | false | Create PR after all steps complete |

## Learning Directory (auto-generated)

Coordinator skills automatically create `.ai/learning/` to store run metrics and patterns. No config needed.

```
.ai/learning/
├── raw/
│   ├── runs.jsonl       # Run records from dx-req-all, dx-step-all, dx-bug-all
│   ├── fixes.jsonl      # Fix patterns from dx-step-all
│   └── bugs.jsonl       # Bug patterns from dx-bug-all
└── fixes.md             # (future) Human-readable fix summary
```

Promoted patterns are written to `.claude/rules/learned-fix-*.md` — these are auto-loaded by Claude on every run.

## Pipeline Environment Variables

These environment variables are set on ADO pipeline runs and read by skills at runtime. They are NOT part of `config.yaml`.

### All CLI Pipelines

| Variable | Description |
|----------|-------------|
| `DX_PIPELINE_MODE` | Set to `"true"` in all CLI pipelines. Enables automatic cross-repo delegation and other pipeline-only behaviors. |
| `ANTHROPIC_API_KEY` | Claude API key for Claude Code CLI authentication. |
| `ADO_ORG_URL` | ADO org URL (e.g. `https://yourorg.visualstudio.com`). Used for plugin marketplace auth and cross-repo delegation. |
| `ADO_ORG_NAME` | Short ADO org name (e.g. `myorg`). Read by `pipeline-agent.js` for org identification. Falls back to `"myorg"` if not set. |
| `DX_MARKETPLACE_URL` | Git URL with ref for the plugin marketplace repo. Only used when `dx-aem-flow/` is not available locally (cross-repo pipelines). |

### Code-Writing Pipelines (BugFix, DevAgent, DoD-Fix)

| Variable | Description |
|----------|-------------|
| `SOURCE_REPO_NAME` | Set to `$(Build.Repository.Name)`. Skills compare this against cross-repo scope to decide delegation. |
| `CROSS_REPO_PIPELINE_MAP` | JSON mapping repo names to pipeline IDs: `{"My-Backend-Repo":"789"}`. Used by the delegation YAML step. |

### delegate.json (cross-repo output)

Written by Claude to `.ai/run-context/delegate.json` when delegation is needed:

```json
{
  "targetRepo": "My-Backend-Repo",
  "pipelineId": "789",
  "reason": "Bug affects backend components",
  "templateParameters": {
    "bugId": "12345",
    "eventId": "evt-001"
  }
}
```

The post-Claude YAML step reads this file and queues the target pipeline via ADO REST API.

## Precedence Rules

For prompt/rule content:
```
.ai/rules/<topic>.md  >  config.yaml overrides:  >  plugin defaults (rules/*.md)
```

For project-specific fields:
```
User-provided URL values  >  config.yaml values  >  auto-detected defaults
```
