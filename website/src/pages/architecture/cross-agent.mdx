      spawn isolated subagents. Coordinator skills use <code>agent:</code> dispatch frontmatter
      for delegation.
    </ContentCard>

    <ContentCard icon="mdi:monitor" iconColor="bg-cyan" title="Copilot CLI">
      Auto-discovers plugin skills directly -- no copying to <code>.github/skills/</code> needed.
      Coordinator skills become Copilot agents with <code>handoffs:</code> for orchestration
      (parsing works since v0.0.402, execution pending).
    </ContentCard>
  </div>
</Section>

<SectionHeading
  badge="Hooks"
  badgeColor="warning"
  title="Unified Hook System"
  subtitle="A single hooks.json file works across both Claude Code and Copilot CLI."
  bg="alt"
/>
<Section>
  <div class="grid grid-cols-1 md:grid-cols-3 gap-6">
    <ContentCard icon="mdi:webhook" iconColor="bg-red-500" title="SessionStart">
      Validates <code>.ai/config.yaml</code> exists and Node version matches <code>.nvmrc</code>.
      Runs on both platforms.
    </ContentCard>

    <ContentCard icon="mdi:shield" iconColor="bg-amber-500" title="PreToolUse">
      Blocks commits on protected branches (main, master, development, develop). Branch guard
      deployed to both plugin hooks and <code>.github/hooks/hooks.json</code>.
    </ContentCard>

    <ContentCard icon="mdi:stop-circle" iconColor="bg-cyan" title="Stop">
      Session stop guard -- checks for secrets, abandoned steps, and uncommitted changes.
      Prompts for confirmation before terminating.
    </ContentCard>
  </div>

  <HighlightBox severity="info" title="Platform Separation">
    Plugin <code>hooks/hooks.json</code> is active in Claude Code only. <code>.github/hooks/hooks.json</code>
    is active in Copilot CLI only. Both use the same <code>version: 1</code> format with PascalCase events.
    <code>/dx-init</code> deploys hooks to both locations.
  </HighlightBox>

  <HighlightBox severity="success" title="Hook Merging (Copilot CLI v1.0.11+)">
    Multiple plugins' hooks now <strong>merge</strong> instead of overwriting each other. dx-core contributes
    general hooks (branch guard, Figma screenshots, subagent log) while dx-aem contributes AEM-specific hooks
    (Chrome DevTools screenshots). In Claude Code, plugin hooks always merged. In Copilot CLI, this required v1.0.11+.
    SessionStart <code>additionalContext</code> is also now injected into the conversation context.
  </HighlightBox>
</Section>

    </ContentCard>

    <ContentCard icon="mdi:shield-lock" iconColor="bg-brand-700" title="Permissions">

      auto-approval independently.
    </ContentCard>
  </div>

  <HighlightBox severity="success" title="Summary: Single Source, Dual Output">
    Same plugins, same marketplace, same skills. Rules use dual frontmatter. Hooks use a single format.
    Agent templates generate both Claude Code and Copilot <code>.agent.md</code> files with platform-specific
    tool name transforms.
  </HighlightBox>
