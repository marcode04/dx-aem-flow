<SectionHeading
  badge="Skills"
  badgeColor="secondary"
  title="Skill Authoring"
  subtitle="YAML frontmatter + markdown body = a reusable command."
  bg="primary"
/>
<Section>
  <CommandBlock label="Complete skill frontmatter">{`---
name: my-skill                    # Required. Becomes /plugin:my-skill command
description: What it does...      # Required. Used for auto-invocation matching
argument-hint: "<expected args>"  # Optional. UI placeholder text
context: fork                     # Optional. "fork" = isolated subagent execution
agent: my-agent                   # Optional. Agent for the fork
disable-model-invocation: false   # Optional. Prevent auto-invocation
---`}</CommandBlock>

  <div class="grid md:grid-cols-3 gap-4 mt-6">
    <ContentCard icon="mdi:play-circle" iconColor="bg-brand-700" title="Inline Skill">
      No <code>context:</code> or <code>agent:</code>. Runs directly in the user's conversation. Good for most skills.
    </ContentCard>
    <ContentCard icon="mdi:source-fork" iconColor="bg-cyan" title="Forked Skill">
      Uses <code>context: fork</code> + <code>agent:</code>. Creates an isolated subagent. Main context stays clean from large MCP output.
    </ContentCard>
    <ContentCard icon="mdi:sitemap" iconColor="bg-emerald-600" title="Coordinator Skill">
      Uses Task tool in the body. Delegates to executor agents. Should ONLY delegate -- never implement directly.
    </ContentCard>
  </div>

  <HighlightBox severity="warning" title="$ARGUMENTS">
    <code>$ARGUMENTS</code> is replaced with whatever the user typed after the command. Example: <code>/dx-req 2416553</code> sets <code>$ARGUMENTS = "2416553"</code>.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Agents"
  badgeColor="warning"
  title="Agent Authoring"
  subtitle="Define who does the work -- model tier, tools, and permissions."
  bg="alt"
/>
<Section>
  <CommandBlock label="Claude Code agent frontmatter">{`---
name: my-agent
description: What it does
model: sonnet              # sonnet | opus | haiku | inherit
tools: Read, Write, Bash   # OMIT to inherit ALL tools (recommended for MCP)
disallowedTools: Write     # Block specific tools while inheriting rest
permissionMode: default    # default | acceptEdits | bypassPermissions
maxTurns: 30
memory: project            # user | project | local
isolation: worktree        # Isolated git worktree
---`}</CommandBlock>

  <HighlightBox severity="error" title="The #1 Gotcha: Tool Inheritance">
    If you specify <code>tools:</code>, the agent can ONLY use those listed tools. MCP tools are BLOCKED unless explicitly listed. Either <strong>omit tools: entirely</strong> to inherit everything (recommended for MCP-dependent agents), or use <code>disallowedTools:</code> to block specific tools while keeping everything else.
  </HighlightBox>

  <h3 class="text-base font-semibold text-brand-900 mt-6 mb-3">Model Tier Strategy</h3>
  <div class="grid md:grid-cols-3 gap-4">
    <ContentCard
      icon="mdi:star"
      iconColor="bg-red-500"
      title="Opus -- Deep Reasoning"
      tags={[{ label: 'T1', color: 'error' }]}
    >
      Architecture, planning, code review, security analysis, root cause diagnosis. Cost: ≈15x haiku.
    </ContentCard>
    <ContentCard
      icon="mdi:diamond-stone"
      iconColor="bg-amber-500"
      title="Sonnet -- Standard Work"
      tags={[{ label: 'T2', color: 'warning' }]}
    >
      Implementation, MCP coordination, PR review, AEM inspection. Cost: ≈5x haiku.

    <ContentCard

    >
      File lookups, doc search, git commits, config reads. Cost: 1x (baseline).
    </ContentCard>
  </div>
</Section>

<SectionHeading

  badgeColor="info"
  title="File Structure and Naming"
  subtitle="Where files live and how they are discovered."
  bg="primary"
/>
<Section>
  <CommandBlock label="Plugin directory layout">{`plugin/
├── .claude-plugin/plugin.json   # Plugin manifest (both platforms)
├── .mcp.json                    # MCP server config
├── agents/                      # Agent definitions (*.md with YAML frontmatter)
├── skills/                      # Skill directories (*/SKILL.md)
├── rules/                       # Default prompt templates
├── hooks/                       # Plugin hooks (hooks.json)
├── data/                        # Seed files copied to project by init
├── shared/                      # Reference files read by skills
└── templates/                   # Init-time file templates
    ├── rules/                   #   Convention rules -> .claude/rules/
    ├── instructions/            #   Detailed docs -> .github/instructions/
    └── agents/                  #   Copilot agents -> .github/agents/`}</CommandBlock>

  <div class="grid md:grid-cols-2 gap-4 mt-6">
    <ContentCard icon="mdi:file-document" iconColor="bg-brand-700" title="Agent File Naming">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><strong>Claude Code:</strong> <code>agents/dx-code-reviewer.md</code></li>
        <li><strong>Copilot (.github):</strong> <code>DxCodeReview.agent.md</code></li>
        <li><strong>Copilot CLI (plugin):</strong> Reads Claude format directly</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-cyan-dark" title="Plugin Manifest">
      <strong>Never</strong> add <code>agents</code> or <code>skills</code> fields for default directories -- this breaks Claude Code with a Zod validation error. Both platforms auto-discover <code>agents/</code> and <code>skills/</code> when these fields are omitted.
    </ContentCard>
  </div>
</Section>

  <HighlightBox severity="info" title="Three-Layer Override System">
    <code>.ai/rules/&lt;topic&gt;.md</code> (full replacement) &gt; <code>config.yaml overrides:</code> (quick tweaks) &gt; plugin defaults (<code>rules/&#42;.md</code>). Projects can also shadow entire skills by creating <code>.claude/skills/&lt;name&gt;/SKILL.md</code>.
  </HighlightBox>
