  <HighlightBox severity="error" title="Plugin MCP Prefix">
    MCP servers in a plugin's <code>.mcp.json</code> get the full prefix: <code>{"mcp__plugin_<plugin-name>_<server>__<tool>"}</code>. Using the shorthand (<code>{"mcp__AEM__"}</code> instead of <code>{"mcp__plugin_dx-aem_AEM__"}</code>) causes "tool not found" failures.
  </HighlightBox>

  <h3 class="text-base font-semibold text-brand-900 mt-6 mb-3">Hook Events</h3>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:shield-check" iconColor="bg-brand-700" title="Pre-Tool Hooks">
      <code>PreToolUse</code> fires before any tool invocation. Use to block dangerous operations or modify input. The branch-guard hook blocks commits on protected branches.
    </ContentCard>
    <ContentCard icon="mdi:check-circle" iconColor="bg-emerald-600" title="Post-Tool Hooks">
      <code>PostToolUse</code> fires after tool completion. Use for auto-formatting, logging, or result processing. <code>PostToolUseFailure</code> fires on tool errors.
    </ContentCard>
  </div>

<SectionHeading
  badge="Debug"
  badgeColor="warning"
  title="Debugging Playbook"
  subtitle="Common issues and how to fix them."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:magnify" iconColor="bg-red-500" title="Skill Not Found">
      <ol class="mt-2 text-sm list-decimal pl-4 space-y-1">
        <li>Check file is at correct skills path</li>
        <li>Verify <code>name:</code> in frontmatter matches what you are typing</li>
        <li>Check <code>description:</code> contains trigger phrases for auto-invocation</li>
        <li>Verify no trailing spaces on <code>---</code> delimiters</li>
      </ol>
    </ContentCard>
    <ContentCard icon="mdi:connection" iconColor="bg-amber-500" title="MCP Tools Not Available">
      <ol class="mt-2 text-sm list-decimal pl-4 space-y-1">
        <li>Check server is running</li>
        <li>Verify config in <code>.mcp.json</code> or <code>.claude/settings.json</code></li>
        <li>If agent, check that <code>tools:</code> is NOT set (blocks MCP inheritance)</li>
        <li>Run <code>ToolSearch("+servername")</code> to load deferred tools</li>
      </ol>
    </ContentCard>
    <ContentCard icon="mdi:eye-off" iconColor="bg-cyan-dark" title="Copilot Agent Not Appearing">
      <ol class="mt-2 text-sm list-decimal pl-4 space-y-1">
        <li>File must have <code>.agent.md</code> extension</li>
        <li>Must be in <code>.github/agents/</code> directory</li>
        <li>Check <code>user-invokable:</code> is not <code>false</code></li>
        <li>Restart VS Code after adding new agents</li>
        <li>Use <code>/troubleshoot</code> to inspect loaded customizations</li>
      </ol>
    </ContentCard>
    <ContentCard icon="mdi:robot-off" iconColor="bg-brand-700" title="Deferred vs Pre-loaded MCP">
      <ol class="mt-2 text-sm list-decimal pl-4 space-y-1">
        <li>If <code>tools:</code> is omitted, MCP tools are <strong>deferred</strong> -- must ToolSearch before calling</li>
        <li>If <code>tools:</code> lists MCP tools explicitly, they are <strong>pre-loaded</strong> -- ToolSearch returns nothing</li>
        <li>Always try calling the tool directly first, fall back to ToolSearch</li>
      </ol>
    </ContentCard>
  </div>

  <HighlightBox severity="warning" title="The #2 Gotcha: ToolSearch False Negatives">
    If an agent has MCP tools explicitly listed in <code>tools:</code>, those tools are pre-loaded and ToolSearch will return nothing for them. This leads agents to wrongly conclude MCP is unavailable. Always try calling the tool directly first.
  </HighlightBox>
</Section>
