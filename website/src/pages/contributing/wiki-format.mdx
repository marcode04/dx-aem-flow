<Section>
  <CommandBlock label="DoR wiki page template (checkbox format)">{`# Definition of Ready -- Checklist

> **Who validates:** Tech Lead / DoR Agent
> **When:** Before "Ready for Development"

## 1. Section Name

- [ ] **Criterion name** \`required\` — What the agent checks
- [ ] **Another criterion** \`recommended\` — Advisory check
- [ ] **Human check** \`human\` — Only human can verify

> **Skip:** When this section can be skipped.

## 2. Next Section

...

## Scoring

- All \`required\` pass -> **Ready for Development**
- 1-2 \`required\` fail -> **Can proceed -- expect clarification**
- 3+ \`required\` fail -> **Needs more detail**`}</CommandBlock>

  <CommandBlock label="DoD wiki page template (table format)">{`# Definition of Done -- Checklist

> **Who validates:** DoD Agent / Tech Lead
> **When:** Before PR merge

## 1. Section Name

| Criterion | Who checks | What to verify |
|-----------|-----------|----------------|
| Criterion name | Agent | What the agent checks |
| Another criterion | Human | What a human verifies |

**Skip trigger:** When this section can be skipped.

## Scoring

**Scoring:**
- All automated criteria pass -> pass verdict
- 1-2 automated failures -> partial verdict
- 3+ automated failures -> fail verdict`}</CommandBlock>
</Section>

<SectionHeading
  badge="Format"
  badgeColor="warning"
  title="Required Elements"
  subtitle="Section headings, criteria tables, and scoring rules."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:format-header-pound" iconColor="bg-brand-700" title="Section Headings">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li>Use <code>## N. Section Title</code> format (numbered, with period)</li>
        <li>Sections are parsed in order -- numbering defines the sequence</li>
        <li>Non-numbered <code>##</code> headings (like <code>## Completion Summary</code>) are metadata, not checklist sections</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:checkbox-marked-outline" iconColor="bg-cyan" title="Criteria Checkboxes">
      Each section contains checkboxes in this format:
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><code>- [ ] **Name** `tag` — Description</code></li>
        <li><strong>Name</strong> -- bold criterion name</li>
        <li><strong>Tag</strong> -- inline code: <code>required</code>, <code>recommended</code>, or <code>human</code></li>
        <li><strong>Description</strong> -- what the agent checks or the human verifies</li>
      </ul>
    </ContentCard>
  </div>

  <h3 class="text-base font-semibold text-brand-900 mt-6 mb-3">Tag Values</h3>
  <div class="overflow-x-auto">
    <table class="w-full text-sm border-collapse">
      <thead>
        <tr class="border-b border-gray-200">
          <th class="text-left py-2 px-4 font-semibold">Tag</th>
          <th class="text-left py-2 px-4 font-semibold">Meaning</th>
          <th class="text-left py-2 px-4 font-semibold">Skill Behavior</th>
        </tr>
      </thead>
      <tbody>
        <tr class="border-b border-gray-100"><td class="py-2 px-4"><code>required</code></td><td class="py-2 px-4">Automated check -- must pass</td><td class="py-2 px-4">Evaluates programmatically -- PASS or FAIL</td></tr>
        <tr class="border-b border-gray-100"><td class="py-2 px-4"><code>recommended</code></td><td class="py-2 px-4">Automated check -- advisory</td><td class="py-2 px-4">WARN at most, never FAIL</td></tr>
        <tr class="border-b border-gray-100"><td class="py-2 px-4"><code>human</code></td><td class="py-2 px-4">Requires human judgment</td><td class="py-2 px-4">Always WARN -- agent cannot verify</td></tr>
      </tbody>
    </table>
  </div>
</Section>

  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:skip-next" iconColor="bg-amber-500" title="Skip Triggers">
      Optional blockquotes after criteria define when a section can be skipped (marked SKIP). DoR uses <code>&gt; **Skip:** ...</code>, DoD uses <code>**Skip trigger:** ...</code>. The text is parsed as natural language. Common patterns:
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li>"Pure configuration or content-only changes"</li>
        <li>"Backend-only or infrastructure changes"</li>
        <li>"Change type is X or Y"</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:counter" iconColor="bg-emerald-600" title="Scoring Rules">
      Skills look for a <code>&#42;&#42;Scoring:&#42;&#42;</code> block with bullet points defining verdicts:
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li>"All automated criteria pass" -- pass verdict</li>
        <li>"1-2 automated failures" -- partial verdict</li>
        <li>"3+ automated failures" -- fail verdict</li>
      </ul>
    </ContentCard>
  </div>
</Section>

  bg="alt"
/>
<Section>
  <ContentCard icon="mdi:clipboard-check" iconColor="bg-brand-700" title="Detection Patterns">
    The hint text after <code>—</code> in each checkbox describes what the skill checks:
    <ul class="mt-2 text-sm list-none p-0 space-y-1">
      <li><code>Clear: [Brand/Market] Verb + Feature</code> -- check title matches pattern</li>
      <li><code>Direct link with node-id</code> -- look for Figma URL with node-id parameter</li>
      <li><code>Field table or "no dialog changes"</code> -- look for table pattern or explicit skip</li>
    </ul>
    <p class="text-sm mt-2">The agent interprets hints using its understanding of the story content. Hints are guidance, not rigid regex patterns.</p>
  </ContentCard>

  <CommandBlock label="Minimal DoR section example">{`## 1. Story Basics

- [ ] **Title and description** \`required\` — Title non-empty, Description non-empty
- [ ] **Acceptance criteria** \`required\` — AC section exists with at least 1 testable condition`}</CommandBlock>
</Section>

        <li>Skills fetch the latest wiki content on every run (no caching)</li>

        <li>Removing criteria removes them from the check -- no code changes needed</li>
      </ul>
    </ContentCard>

      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><strong>DoR:</strong> Checkbox format must be exact: <code>- [ ] **Name** `tag` — Description</code></li>
        <li><strong>DoD:</strong> Table format must be exact: pipe-separated, header + separator + data rows</li>
        <li>Use <code>## N. Title</code> format for checklist sections (numbered with period)</li>
        <li>Non-numbered headings are treated as metadata, not sections</li>
      </ul>

<SectionHeading
  badge="Troubleshooting"
  badgeColor="error"
  title="Troubleshooting"
  subtitle="Common issues when skills cannot parse wiki pages."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <HighlightBox severity="error" title='"wiki-dor-url not configured"'>
      Add the URL to <code>.ai/config.yaml</code> under the <code>scm:</code> section. Both <code>wiki-dor-url</code> and <code>wiki-dod-url</code> are required.
    </HighlightBox>
    <HighlightBox severity="warning" title="Skill does not recognize a criterion">
      For DoR pages, check that the checkbox format is exact: <code>- [ ] **Name** `tag` — Description</code>. For DoD pages, check that the table has three columns, pipe-separated, with the header row matching <code>Criterion | Who checks | What to verify</code>.
    </HighlightBox>
    <HighlightBox severity="warning" title="New section not being checked">
      Ensure the heading uses the <code>## N. Title</code> format (numbered, with period after the number). Non-numbered headings are metadata.
    </HighlightBox>
    <HighlightBox severity="info" title='"Could not fetch wiki page"'>
      Verify the URL is correct -- it should be a full URL including the wiki identifier and page path.
    </HighlightBox>
  </div>
</Section>
