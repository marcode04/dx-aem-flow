  </CommandBlock>

  <HighlightBox severity="warning" title="Multi-Agent Orchestration Under the Hood">
    The coordinator invokes each phase skill directly via Skill() calls -- a composable
    pattern applied to AI agents. Each skill runs in its own isolated context window,
    performing deep repository intelligence without polluting the orchestrator's context. Only a compact
    summary propagates upward. This is <strong>context engineering</strong> in practice.
  </HighlightBox>

  <div class="flex flex-col gap-4 mt-6">
    <ContentCard icon="mdi:magnify" iconColor="bg-brand-700" title="Phase 1: Extract — /dx-figma-extract" horizontal>
      Live <strong>tool use</strong> via Figma MCP -- pulls the node tree, design tokens, and screenshots
      directly from the running Figma desktop app. Classifies each layer using adaptive intelligence.
      Downloads image assets locally.

      <p class="mt-1 text-[#7a7a8a] font-mono text-sm">Output: figma-extract.md + figma-reference.png + assets/</p>
    </ContentCard>

    <ContentCard icon="mdi:account-group" iconColor="bg-cyan" title="Phase 2: Prototype — /dx-figma-prototype" horizontal>
      <strong>Parallel reasoning</strong> across two specialized agents: <code>dx-figma-styles</code> performs
      repository intelligence on your CSS/SCSS conventions while <code>dx-figma-markup</code> simultaneously
      discovers HTML/accessibility patterns. The orchestrator merges both results into a standalone prototype
      that uses your project's actual design tokens.

      <p class="mt-1 text-[#7a7a8a] font-mono text-sm">Output: figma-conventions.md + prototype/index.html + styles.css + script.js</p>
