<Section>

      Daily cap per agent -- prevents runaway execution. Each agent type has its own limit tuned to expected workload.
    </HighlightBox>
    <HighlightBox severity="error" title="Token Budget">
      Monthly cap with alerts at 80%, degrades at 90%, halts at 100%. No surprise bills.
    </HighlightBox>
    <HighlightBox severity="info" title="Policy Gates">
      Agents can only do what their role allows. A PR reviewer cannot merge. A doc agent cannot modify code.
    </HighlightBox>
    <HighlightBox severity="success" title="Deduplication">
      Duplicate webhook events are silently dropped. No double-processing, no wasted tokens.
    </HighlightBox>

<SectionHeading
  badge="Engineering"
  title="Best Practices & Innovative Ideas"
  subtitle="Production-grade patterns that make this enterprise-ready -- not a prototype, not a demo, but a system built for real teams shipping real code."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:format-list-checks" iconColor="bg-cyan" title="Templates for Predictive Outcome" horizontal>
      Every skill outputs to a structured template -- explain.md, implement.md, research.md. Templates encode domain knowledge and guarantee consistent output.
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-amber-500" title="Hooks to Offload LLM Usage" horizontal>
      Shell hooks intercept tool calls before the LLM sees them. Figma screenshot hooks auto-cache assets. Pre-commit hooks validate without burning tokens.
    </ContentCard>
    <ContentCard icon="mdi:brain" iconColor="bg-brand-700" title="Subagents to Protect Context" horizontal>
      Heavy operations run in isolated subagents that return compact summaries. The coordinator's context window stays clean. Right model for the job.
    </ContentCard>
    <ContentCard icon="mdi:lightning-bolt" iconColor="bg-emerald-600" title="Direct Skill Invocation -- No Middleman" horizontal>
      Coordinators invoke skills directly via Skill() calls. No intermediary agent needed. Self-healing with 3 layers: fix, heal, retry loop.
    </ContentCard>
    <ContentCard icon="mdi:lock" iconColor="bg-red-500" title="Explicit Tool Allowlists" horizontal>
      Every agent declares exactly which tools it can use. A PR reviewer cannot run Bash. Optional tools are discovered at runtime via ToolSearch.
    </ContentCard>
    <ContentCard icon="mdi:link" iconColor="bg-cyan-dark" title="Convention-Based Discovery" horizontal>
      Skills write to predictable paths: .ai/specs/&lt;id&gt;-&lt;slug&gt;/explain.md. The next skill finds its predecessor's output by convention.
    </ContentCard>
    <ContentCard icon="mdi:folder" iconColor="bg-red-500" title="The .ai/ Folder -- Clean Separation" horizontal>
      All AI artifacts live in .ai/ -- specs, config, templates, rules, learning data. Project source stays untouched.
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-brand-700" title="Config-Driven, Never Hardcoded" horizontal>
      One config.yaml holds all project values. Same plugin codebase runs across 4 repos and 50+ potential projects without a code change.
    </ContentCard>
    <ContentCard icon="mdi:layers" iconColor="bg-amber-500" title="Three-Layer Override System" horizontal>
      Plugin defaults, .ai/rules/ domain overrides, .claude/rules/ permanent conventions. Teams customize behavior by adding a file.
    </ContentCard>
    <ContentCard icon="mdi:memory" iconColor="bg-red-500" title="Memory via Markdown -- Between Steps" horizontal>
      Each skill writes findings to a .md file. No database, no API, no shared memory bus -- just files on disk. Survives session crashes.
    </ContentCard>
    <ContentCard icon="mdi:shield-check" iconColor="bg-cyan-dark" title="Subagent Memory Isolation" horizontal>
      Subagents run in their own context window. The parent never sees the 500 files the subagent searched. Context cost doesn't propagate upward.
    </ContentCard>
    <ContentCard icon="mdi:swap-horizontal" iconColor="bg-emerald-600" title="Cross-Repo Intelligence" horizontal>
      Frontend bug that needs a backend fix? The agent detects the cross-repo dependency and delegates to the correct repository's pipeline.
    </ContentCard>
    <ContentCard icon="mdi:target" iconColor="bg-cyan" title="Model Tier Strategy" horizontal>
      Opus for deep code review. Sonnet for execution. Haiku for simple lookups. Cost scales with complexity, not volume.
    </ContentCard>
    <ContentCard icon="mdi:hammer-wrench" iconColor="bg-brand-700" title="Dual-IDE: Claude Code + GitHub Copilot" horizontal>
      Skills written for Claude Code, auto-discovered by Copilot CLI. Agent templates generated for Copilot orchestration. Core workflows work in both IDEs.
    </ContentCard>
  </div>
