<SectionHeading
  badge="Concept"
  title="What is an Agent?"
  subtitle="A markdown file defining a specialized AI persona -- what model, which tools, what role."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:robot" iconColor="bg-brand-700" title="Personas, Not Prompts">
      While skills tell the AI <strong>what</strong> to do, agents define <strong>who</strong> does
      it. Think of agents as team members with different expertise. A code reviewer agent uses Opus
      for deep reasoning. A file lookup agent uses Haiku for speed. Each agent has exactly the tools
      it needs -- nothing more.
    </ContentCard>
    <ContentCard icon="mdi:arrow-decision" iconColor="bg-cyan" title="The Key Difference: Isolation">
      A skill runs <strong>inline</strong> in your main conversation -- same AI, same context window,
      same tools. An agent runs in a <strong>separate context</strong> -- fresh context window, its own
      system prompt, possibly a different model. The agent does its work independently and returns a
      summary. This is why agents are used for heavy tasks that would bloat your main conversation.
    </ContentCard>
  </div>
</Section>

  <HighlightBox severity="info" title="The Orchestration Pattern">
    In practice, <strong>skills orchestrate and agents execute</strong>.
    When you type <code>/dx-plan</code>, the skill loads into the main conversation.
    The skill may then spawn a <code>dx-code-reviewer</code> agent (Opus, read-only tools, fresh context)
    to do heavy analysis. The agent returns a summary, and the skill continues with the result.
    Users interact with skills; agents work behind the scenes.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Decision"
  badgeColor="warning"
  title="When to Use Which?"
  subtitle="A quick guide for plugin authors."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:file-document" iconColor="bg-brand-700" title="Create a Skill when...">
      <ul class="mt-2 list-none p-0 text-sm space-y-1">
        <li>The user invokes it directly (<code>/command</code>)</li>
        <li>It's a reusable workflow or process</li>
        <li>It teaches conventions or domain knowledge</li>
        <li>Results should stay in the main conversation</li>
        <li>It bundles scripts or templates</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:robot" iconColor="bg-cyan" title="Create an Agent when...">
      <ul class="mt-2 list-none p-0 text-sm space-y-1">
        <li>You need a different model (Haiku for speed, Opus for depth)</li>
        <li>You need to restrict tools (read-only, no file edits)</li>
        <li>Heavy output would bloat the main context</li>
        <li>Other skills should delegate work to it</li>
        <li>You want parallel execution (multiple agents at once)</li>
      </ul>
    </ContentCard>
  </div>
</Section>
