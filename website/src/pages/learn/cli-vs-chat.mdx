      The AI chat panel built into VS Code. In <strong>Agent Mode</strong>, it can autonomously edit files,
      run terminal commands, and iterate until a task is complete. It uses VS Code's language server
      for deep code understanding (symbols, references, diagnostics).
      <CommandBlock label="How to access">VS Code → Chat panel → select "Agent" mode</CommandBlock>
    </ContentCard>
    <ContentCard icon="mdi:console" iconColor="bg-brand-700" title="vs Copilot CLI">
      <strong>Chat panel</strong> = visual, integrated with VS Code UI, accesses editor context (open files, selections, diagnostics).

      <strong>Copilot CLI</strong> = terminal-based, supports plugins/skills/hooks/MCP natively, sessions persist independently.

      You can start a Copilot CLI session <em>from</em> the Chat panel and hand off conversations between them.
    </ContentCard>
  </div>
  <HighlightBox severity="info" title="File Discovery -- Three Platforms">
    <strong>VS Code Chat</strong> reads <code>.github/agents/</code> natively, <code>.claude/rules/</code> via <code>chat.instructionsFilesLocations</code>,
    and skills via <code>chat.agentSkillsLocations</code> (defaults include <code>.claude/skills/</code>).
    MCP servers come from <code>.vscode/mcp.json</code>.

    <strong>Copilot CLI</strong> reads <code>.github/agents/</code> natively, <code>.claude/rules/</code> via <code>COPILOT_CUSTOM_INSTRUCTIONS_DIRS</code> env var,
    and plugin skills automatically.

    <strong>Claude Code</strong> reads everything from <code>.claude/</code> natively plus plugin agents, skills, hooks, and MCP servers.
  </HighlightBox>
</Section>

  <HighlightBox severity="warning" title="Cross-Platform Gotchas">
    <strong>Tool names differ:</strong> Copilot CLI uses <code>read, edit, search</code>. VS Code Chat uses <code>codebase, editFiles, runCommands</code>.
    Agent templates include <strong>both</strong> -- unrecognized names are ignored on each platform.

    <strong>Spelling:</strong> Claude Code uses <code>user-invocable</code>, Copilot CLI uses <code>user-invokable</code> (different spelling!).

    <strong>MCP config:</strong> Claude Code / Copilot CLI read <code>.mcp.json</code> (<code>mcpServers:</code>).
    VS Code Chat reads <code>.vscode/mcp.json</code> (<code>servers:</code>). Different file, different root key.
  </HighlightBox>

<Section>
  <HighlightBox severity="info" title="Key Insight">
    Claude Code and Copilot CLI are <strong>nearly identical in capability</strong> -- both are terminal-based, both run the same {stats.totalSkills} skills, both load plugins and MCP servers.
    The real difference is between <strong>terminal CLIs</strong> (Claude Code + Copilot CLI) and <strong>VS Code Chat</strong> (IDE-integrated, visual, different file discovery).
  </HighlightBox>

  <div class="grid md:grid-cols-3 gap-4 mt-4">
    <ContentCard icon="mdi:console" iconColor="bg-cyan" title="Terminal CLIs (Claude Code / Copilot CLI)" tags={[{ label: 'Full power', color: 'secondary' }]}>
