<SectionHeading
  badge="Concept"
  title="What is MCP?"
  subtitle="Model Context Protocol -- a standard way for AI to call external APIs."
  bg="alt"
/>
<Section>
  <ContentCard icon="mdi:hub" iconColor="bg-brand-700" title="Direct API Access, Not Guessing">
    Instead of the AI guessing about your AEM instance, it queries it directly. Instead of
    parsing ADO or Jira web pages, it calls the tracker API through MCP. MCP is a standard protocol --
    any tool that implements it can be used by any AI that supports it.
  </ContentCard>
</Section>

      title="AEM MCP"
      tags={[{ label: 'aem-mcp-server', color: 'warning' }]}
    >
      Live JCR access — components, dialogs, pages, node queries. Multi-instance support (local + QA). Included with the dx-aem plugin; also available standalone via <a href="https://www.npmjs.com/package/aem-mcp-server">npm</a>.
      <CommandBlock label="Tools">{`getNodeContent, scanPageComponents,
createPage, updateComponent`}</CommandBlock>
    </ContentCard>
    <ContentCard
      icon="mdi:earth"
      iconColor="bg-cyan-dark"
      title="Chrome DevTools MCP"
      tags={[{ label: 'stdio', color: 'info' }]}
    >

      verification of AEM components.
      <CommandBlock label="Tools">{`navigate_page, take_screenshot,
click, evaluate_script`}</CommandBlock>
    </ContentCard>
    <ContentCard
      icon="mdi:palette"
      iconColor="bg-cyan"
      title="Figma MCP"
      tags={[{ label: 'HTTP', color: 'secondary' }]}
    >
      Design extraction -- tokens, screenshots, component context. Powers the
      Figma-to-code pipeline.
      <CommandBlock label="Tools">{`get_design_context, get_screenshot,
get_variable_defs`}</CommandBlock>
    </ContentCard>
    <ContentCard
      icon="mdi:format-list-bulleted"

      <CommandBlock label="Tools">{`jira_get_issue, jira_search,
confluence_create_page, jira_add_comment`}</CommandBlock>

    <ContentCard

<SectionHeading
  badge="Flow"
  badgeColor="warning"
  title="How It Works"
  subtitle="Skill instructs AI, AI calls MCP, MCP queries the system, AI reasons about the result."
  bg="alt"
/>
<Section>
  <HighlightBox severity="info" title="Example Flow">
    Skill tells AI "check this component" → AI calls <code>AEM/scanPageComponents</code> →
    MCP server queries AEM JCR → returns JSON → AI reasons about the result and reports findings.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Config"
  badgeColor="info"
  title="Configuration"
  subtitle="Two locations: project-level and plugin-level."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:cog" iconColor="bg-brand-700" title="Project .mcp.json">
      Shared by both Claude Code and Copilot CLI. For servers that apply to the whole project.
      <CommandBlock label=".mcp.json (ADO example)">{`{
  "mcpServers": {
    "ado": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@azure-devops/mcp", "my-org"]
    }
  }
}`}</CommandBlock>
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-cyan" title="Plugin .mcp.json">
      Auto-loaded when the plugin is installed. For servers bundled with a specific plugin
      (AEM, Chrome DevTools, Figma, axe).
      <CommandBlock label="Tool naming convention">{`# Project-level:
mcp__ado__wit_get_work_item

# Plugin-level (dx-aem plugin):
mcp__plugin_dx-aem_AEM__getNodeContent`}</CommandBlock>
    </ContentCard>
  </div>
</Section>
