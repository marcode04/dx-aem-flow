<SectionHeading

  badgeColor="secondary"
  title="MCP Environment Variables"
  subtitle="ADO MCP is auto-configured. Jira and AEM MCPs need env vars."
  bg="alt"
/>
<Section>
  <HighlightBox severity="success" title="ADO MCP — Auto-Configured">
    Browser-based OAuth. On first use, opens a browser to authenticate. No PAT or env var needed.
  </HighlightBox>

  <HighlightBox severity="info" title="Jira/Confluence MCP">
    Set in your shell profile or <code>.claude/settings.local.json</code>:
    <code>JIRA_PERSONAL_TOKEN</code>, <code>JIRA_BASE_URL</code>, <code>JIRA_USER_EMAIL</code>.
  </HighlightBox>

  <HighlightBox severity="warning" title="AEM MCP">
    Requires the <code>AEM_INSTANCES</code> environment variable. Add to your shell profile — both Claude Code and Copilot CLI inherit shell env vars.
  </HighlightBox>

  <CommandBlock label="~/.bashrc or ~/.zshrc">{`export AEM_INSTANCES="local:http://localhost:4502:admin:admin"

# Multiple instances (comma-separated):
# export AEM_INSTANCES="local:http://localhost:4502:admin:admin,qa:https://qa-author.example.com:USER:PASS"`}</CommandBlock>

  <HighlightBox severity="info" title="Format">
    Colon-delimited: <code>name:url:user:pass</code>. Multiple instances separated by commas.
    Alternatively, set in <code>.claude/settings.local.json</code> under <code>"env"</code>.
  </HighlightBox>

  <HighlightBox severity="info" title="Chrome DevTools MCP">
    Auto-launches a Chrome instance via stdio. No manual setup — just ensure Chrome is installed.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Config"
  badgeColor="info"
  title="Configuration Details"
  subtitle="Auto-configured by /dx-init but documented here for manual setup or troubleshooting."
  bg="primary"
/>
<Section>
  <h3 class="text-base font-semibold text-brand-900 mb-4">Co-authored Commit Footer</h3>
  <p class="text-sm text-gray-600 mb-2">Disable Claude's default co-author footer in <code>.claude/settings.json</code>:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "attribution": {
    "commit": "",
    "pr": ""
  }
}`}</CommandBlock>

  <h3 class="text-base font-semibold text-brand-900 mt-8 mb-4">MCP Server Enablement</h3>
  <p class="text-sm text-gray-600 mb-2">Enable only the servers your project uses — unused servers consume tokens:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "enableAllProjectMcpServers": false,
  "enabledMcpjsonServers": ["ado"]
}`}</CommandBlock>
  <p class="text-sm text-gray-500 mt-2">Add <code>"AEM"</code> and <code>"chrome-devtools-mcp"</code> for AEM projects.</p>

  <h3 class="text-base font-semibold text-brand-900 mt-8 mb-4">Permissions</h3>
  <p class="text-sm text-gray-600 mb-2">Start minimal, expand as needed — Claude prompts for anything not in the allow list:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(npx:*)",
      "Bash(mvn*)",
      "mcp__ado__wit_get_work_item",
      "mcp__ado__wit_list_work_item_comments",
      "mcp__ado__search_code"
    ]
  }
}`}</CommandBlock>
  <HighlightBox severity="warning" title="Avoid Wildcards">
    Do not use wildcards like <code>{"mcp__ado__*"}</code> — they send full tool schemas on every request, wasting tokens.
  </HighlightBox>
</Section>

    severity-threshold: 80`}</CommandBlock>
    </ContentCard>
