<SectionHeading
  badge="Step 3"
  badgeColor="secondary"
  title="Configure MCP Environment Variables"
  subtitle="ADO MCP is auto-configured. Jira and AEM MCPs need manual setup."
  bg="primary"
/>
<Section>
  <HighlightBox severity="success" title="ADO MCP -- Auto-Configured">
    The ADO MCP server uses browser-based OAuth. On first use, it opens a browser to authenticate with your ADO account. No PAT or env var needed for regular development.
  </HighlightBox>

  <HighlightBox severity="info" title="Jira/Confluence MCP -- Manual Configuration">
    If using Jira instead of ADO, set the following env vars in <code>.claude/settings.local.json</code> and your shell profile:
    <code>JIRA_PERSONAL_TOKEN</code>, <code>JIRA_BASE_URL</code> (e.g., <code>https://myorg.atlassian.net</code>),
    and <code>JIRA_USER_EMAIL</code>.
  </HighlightBox>

  <HighlightBox severity="warning" title="AEM MCP -- Manual Configuration Required">
    AEM MCP requires <code>AEM_INSTANCES</code> to be set in <strong>two places</strong>:
    <ol class="mt-2 text-sm">
      <li><code>{"~"}/.bashrc</code> or <code>{"~"}/.zshrc</code> -- for Claude Code CLI and Copilot CLI</li>
      <li><code>.claude/settings.local.json</code> -- for Claude Code session env</li>
    </ol>
    <p class="text-xs text-gray-500 mt-2">Why both? Copilot CLI does not read <code>settings.local.json</code> -- it only inherits shell environment variables.</p>
  </HighlightBox>

  <CommandBlock label="~/.bashrc or ~/.zshrc">{`export AEM_INSTANCES='[{"name":"local","url":"http://localhost:4502","auth":"admin:admin"}]'`}</CommandBlock>

  <CommandBlock label=".claude/settings.local.json">{`{
  "env": {
    "AEM_INSTANCES": "[{\\"name\\":\\"local\\",\\"url\\":\\"http://localhost:4502\\",\\"auth\\":\\"admin:admin\\"}]"
  }
}`}</CommandBlock>

  <HighlightBox severity="info" title="Chrome DevTools MCP">
    Chrome DevTools MCP auto-launches a Chrome instance via stdio. No manual setup required -- just ensure Chrome is installed.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Details"
  badgeColor="info"
  title="Configuration Details"
  subtitle="Auto-configured by /dx-init but documented here for manual setup or troubleshooting."
  bg="primary"
/>
<Section>
  <h3 class="text-base font-semibold text-brand-900 mb-4">Co-authored Commit Footer</h3>
  <p class="text-sm text-gray-600 mb-2">Disable Claude's default co-author footer in <code>.claude/settings.json</code>:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "attribution": {
    "commit": "",
    "pr": ""
  }
}`}</CommandBlock>

  <h3 class="text-base font-semibold text-brand-900 mt-8 mb-4">MCP Server Enablement</h3>
  <p class="text-sm text-gray-600 mb-2">Enable only the servers your project uses -- unused servers consume tokens:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "enableAllProjectMcpServers": false,
  "enabledMcpjsonServers": ["ado"]
}`}</CommandBlock>
  <p class="text-sm text-gray-500 mt-2">Add <code>"AEM"</code> and <code>"chrome-devtools-mcp"</code> for AEM projects.</p>

  <h3 class="text-base font-semibold text-brand-900 mt-8 mb-4">Permissions</h3>
  <p class="text-sm text-gray-600 mb-2">Start minimal, expand as needed -- Claude prompts for anything not in the allow list:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(npx:*)",
      "Bash(mvn*)",
      "mcp__ado__wit_get_work_item",
      "mcp__ado__wit_list_work_item_comments",
      "mcp__ado__search_code"
    ]
  }
}`}</CommandBlock>
  <HighlightBox severity="warning" title="Avoid Wildcards">
    Do not use wildcards like <code>{"mcp__ado__*"}</code> -- they send full tool schemas on every request, wasting tokens.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Customization"
  badgeColor="warning"
  title="Customization"
  subtitle="Adjust agent behavior, override config, or shadow entire skills."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-3 gap-4">
    <ContentCard icon="mdi:pencil" iconColor="bg-brand-700" title="Shared Rules">
      Edit files in <code>.ai/rules/</code> to change review criteria, answer tone, or pragmatism filters.
      Changes apply to both local skills and automation agents.

      - `pr-review.md` -- severity, confidence
      - `pr-answer.md` -- response tone
      - `pragmatism.md` -- question filtering
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-cyan" title="Config Overrides">
      Add an <code>overrides:</code> section to <code>.ai/config.yaml</code>:
      <CommandBlock>{`overrides:
  pr-review:
    tone: "direct, no praise"
    severity-threshold: 80
  pr-answer:
    persona: "senior developer"`}</CommandBlock>
    </ContentCard>
    <ContentCard icon="mdi:content-copy" iconColor="bg-amber-500" title="Skill Shadowing">
      Override any plugin skill by creating one with the same name in your project.
      Project-level skills take precedence over plugin skills automatically.
      <CommandBlock>{`mkdir -p .claude/skills/commit
# Write your own SKILL.md`}</CommandBlock>
    </ContentCard>
  </div>
</Section>

<SectionHeading

  subtitle="Set up project structure without Claude Code or Copilot CLI."
  bg="alt"
/>
<Section>
  <p class="text-sm text-gray-600 mb-4">A zero-dependency Node.js utility that replicates <code>/dx-init</code> + <code>/aem-init</code> output:</p>
  <CommandBlock label="Scaffold without AI tools">{`node dx/cli/bin/dx-scaffold.js /path/to/your-project --all`}</CommandBlock>
  <div class="grid md:grid-cols-2 gap-4 mt-4">
    <ContentCard title="Available Flags">
      - `--aem` -- AEM rules + seed data
      - `--copilot` -- Copilot agent templates
      - `--all` -- both AEM and Copilot
      - `--force` -- overwrite existing files
    </ContentCard>
    <HighlightBox severity="info" title="When to Use">
      Use the standalone scaffold when you need to set up project structure in CI/CD pipelines
      or for teammates who have not yet installed Claude Code.
    </HighlightBox>
  </div>
</Section>
