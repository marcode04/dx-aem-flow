  <h3 class="text-base font-semibold text-brand-900 mt-6 mb-3">Enable Only What You Need</h3>
  <p class="text-sm text-gray-600 mb-2">Unused MCP servers consume tokens. Enable selectively in <code>.claude/settings.json</code>:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "enableAllProjectMcpServers": false,
  "enabledMcpjsonServers": ["ado"]
}`}</CommandBlock>
  <p class="text-sm text-gray-500 mt-2">Add <code>"AEM"</code> and <code>"chrome-devtools-mcp"</code> for AEM projects.</p>

  <HighlightBox severity="info" title="Env Vars — Two Options in Claude Code">
    Claude Code can read env vars from your <strong>shell profile</strong> (recommended — works for both Claude Code and Copilot CLI)
    or from <code>.claude/settings.local.json</code> under <code>"env"</code> (Claude Code only, gitignored).
    Shell profile is preferred for portability.
  </HighlightBox>
</Section>

  <h3 class="text-base font-semibold text-white mt-8 mb-3">Permissions</h3>
  <p class="text-sm text-gray-200 mb-2">Start minimal, expand as needed — Claude prompts for anything not in the allow list:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "Bash(npx:*)",
      "Bash(mvn*)",
      "mcp__ado__wit_get_work_item",
      "mcp__ado__wit_list_work_item_comments",
      "mcp__ado__search_code"
    ]
  }
}`}</CommandBlock>
  <HighlightBox severity="warning" title="Avoid Wildcards">
    Do not use wildcards like <code>{"mcp__ado__*"}</code> — they send full tool schemas on every request, wasting tokens.
  </HighlightBox>

  <h3 class="text-base font-semibold text-white mt-8 mb-3">Co-authored Commit Footer</h3>
  <p class="text-sm text-gray-200 mb-2">Disable Claude's default co-author footer:</p>
  <CommandBlock label=".claude/settings.json">{`{
  "attribution": {
    "commit": "",
    "pr": ""
  }
}`}</CommandBlock>
</Section>
