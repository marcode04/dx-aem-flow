<SectionHeading

  title="Environment Variable Limitation"
  subtitle="Critical difference from Claude Code."
  bg="alt"
/>
<Section>
  <HighlightBox severity="error" title="Copilot CLI Does Not Read settings.local.json">
    Copilot CLI only inherits shell environment variables (<code>HOME</code>, <code>TMPDIR</code>, <code>PWD</code>, and custom exports).
    Plugin MCP configs using <code>$&#123;AEM_INSTANCES&#125;</code> work in Claude Code but <strong>fail silently</strong> in Copilot CLI
    unless the variable is exported in your shell profile.
  </HighlightBox>

  <CommandBlock label="~/.bashrc or ~/.zshrc (REQUIRED for Copilot CLI)">{`export AEM_INSTANCES="local:http://localhost:4502:admin:admin"
export AXE_API_KEY="your-axe-api-key"  # optional, for accessibility testing

# Jira (if using Jira instead of ADO)
export JIRA_PERSONAL_TOKEN="your-token"
export JIRA_BASE_URL="https://myorg.atlassian.net"
export JIRA_USER_EMAIL="you@example.com"`}</CommandBlock>
</Section>

<SectionHeading
  badge="MCP"
  badgeColor="warning"
  title="MCP Server Configuration"
  subtitle="Copilot CLI reads MCP config from a different location than Claude Code."
  bg="primary"
/>
<Section>
  <HighlightBox severity="warning" title="Copilot CLI Does Not Read Project .mcp.json">
    Claude Code reads <code>.mcp.json</code> at the project root. Copilot CLI only reads from
    <code>~/.copilot/mcp-config.json</code> (global) or plugin-bundled <code>.mcp.json</code> files.
    Project-specific servers like ADO must be configured manually.
  </HighlightBox>

  <CommandBlock label="~/.copilot/mcp-config.json (create this file)">{`{
  "servers": {
    "ado": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@azure-devops/mcp", "<your-ado-org>"]
    }
  }
}`}</CommandBlock>

  <HighlightBox severity="info" title="Per-Session Alternative">
    For multiple ADO organizations: <code>copilot --additional-mcp-config @.mcp-copilot.json</code>
  </HighlightBox>
