<SectionHeading
  badge="Critical"
  badgeColor="error"
  title="Shell-Only Environment Variables"
  subtitle="The most important difference from Claude Code."
  bg="alt"
/>
<Section>
  <HighlightBox severity="error" title="Copilot CLI Does Not Read settings.local.json">
    Copilot CLI only inherits shell environment variables. Plugin MCP configs using
    <code>{'${AEM_INSTANCES}'}</code> work in Claude Code but <strong>fail silently</strong> in Copilot CLI
    unless the variable is exported in your shell profile (<code>{"~"}/.zshrc</code> or <code>{"~"}/.bashrc</code>).
  </HighlightBox>

  <p class="text-sm text-gray-600 mt-4">
    All env vars from the <a href={`${import.meta.env.BASE_URL}/setup/`}>Setup page</a> must be in your shell profile.
    If you previously set them in <code>.claude/settings.local.json</code>, add them to your shell profile too.
  </p>
</Section>

<SectionHeading
  badge="MCP"
  badgeColor="warning"
  title="MCP Server Configuration"
  subtitle="Copilot CLI reads MCP config from a different location than Claude Code."
  bg="primary"
/>
<Section>
  <HighlightBox severity="warning" title="Copilot CLI Does Not Read Project .mcp.json">
    Claude Code reads <code>.mcp.json</code> at the project root. Copilot CLI only reads from
    <code>{"~"}/.copilot/mcp-config.json</code> (global) or plugin-bundled <code>.mcp.json</code> files.
    Project-specific servers like ADO must be configured manually.
  </HighlightBox>

  <CommandBlock label="~/.copilot/mcp-config.json (create this file)">{`{
  "servers": {
    "ado": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@azure-devops/mcp", "<your-ado-org>"]
    }
  }
}`}</CommandBlock>

  <HighlightBox severity="info" title="Per-Session Alternative">
    For multiple ADO organizations: <code>copilot --additional-mcp-config @.mcp-copilot.json</code>
  </HighlightBox>

<SectionHeading
  badge="v1.0.11"
  badgeColor="success"
  title="What's New in v1.0.11"
  subtitle="Key improvements for plugin-based workflows."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:folder-network" iconColor="bg-emerald-600" title="Monorepo Discovery">
      Skills, agents, MCP servers, and custom instructions are now discovered at <strong>every directory level</strong>
      from the working directory up to the git root. Users in <code>monorepo/apps/frontend/</code> automatically
      find <code>.github/agents/</code> installed at the repo root by <code>/dx-init</code>.
    </ContentCard>
    <ContentCard icon="mdi:merge" iconColor="bg-cyan" title="Hook Merging">
      Hooks from multiple extensions now <strong>merge</strong> instead of overwriting. dx-core and dx-aem can each
      contribute their own hooks without conflict. SessionStart <code>additionalContext</code> is injected into the
      conversation context.
    </ContentCard>
    <ContentCard icon="mdi:account" iconColor="bg-brand-700" title="Personal Skills Directory">
      <code>{"~"}/.agents/skills/</code> is now a personal skill discovery directory (aligning with VS Code Chat).
      Use it for individual developer customizations that don't belong in the shared repo.
    </ContentCard>
    <ContentCard icon="mdi:swap-horizontal" iconColor="bg-amber-500" title="Session Management">
      <code>/clear</code> abandons the session entirely. <code>/new</code> starts a fresh conversation while keeping
      the old session backgrounded. Both accept an optional prompt. <code>/cd</code> now keeps a separate working
      directory per session, restored when switching.
    </ContentCard>
  </div>

  <HighlightBox severity="info" title="Atlassian MCP OAuth">
    MCP OAuth authentication now works with the Atlassian Rovo MCP Server (Dynamic Client Registration with
    non-standard auth metadata URLs). Teams using Jira/Confluence via MCP should see smoother setup.
  </HighlightBox>
</Section>
