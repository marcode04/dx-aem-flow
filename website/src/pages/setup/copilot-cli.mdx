<SectionHeading
  badge="Step 3"
  badgeColor="secondary"
  title="Configure MCP Environment Variables"
  subtitle="Critical difference: Copilot CLI only reads shell environment variables."
  bg="primary"
/>
<Section>
  <HighlightBox severity="error" title="Copilot CLI Environment Variable Limitation">
    Copilot CLI does <strong>not</strong> expand custom env vars from config files -- only <code>HOME</code>, <code>TMPDIR</code>, <code>PWD</code>. Plugin MCP configs using <code>$&#123;AEM_INSTANCES&#125;</code> work in Claude Code but <strong>fail silently</strong> in Copilot CLI unless the variable is set in your shell profile.
  </HighlightBox>

  <CommandBlock label="~/.bashrc (REQUIRED for Copilot CLI -- AEM projects only)">{`export AEM_INSTANCES="local:http://localhost:4502:admin:admin,qa:https://qa-author.example.com:USER:PASS"
export AXE_API_KEY="your-axe-api-key"  # optional, for accessibility testing`}</CommandBlock>

  <HighlightBox severity="info" title="After Editing Shell Profile">
    Restart your terminal or run <code>source ~/.bashrc</code> / <code>source ~/.zshrc</code> for changes to take effect.
  </HighlightBox>
</Section>
