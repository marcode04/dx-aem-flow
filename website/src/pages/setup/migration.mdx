<SectionHeading
  badge="Step 1"
  badgeColor="warning"
  title="Remove Old Standalone Files"
  subtitle="Clear files that will be reinstalled fresh by the plugins."
  bg="alt"
/>
<Section>
  <HighlightBox severity="error" title="Keep Project-Specific Files">
    Do NOT remove <code>.ai/automation/infra.json</code>, <code>.ai/automation/repos.json</code>, <code>.ai/specs/</code>, <code>.ai/me.md</code>, or <code>.claude/CLAUDE.md</code>. These contain real project data that must survive the migration.
  </HighlightBox>

  <CommandBlock label="Remove plugin-managed files">{`# Automation source code -- plugin installs updated versions
rm -rf .ai/automation/lambda \\
       .ai/automation/pipelines \\
       .ai/automation/agents \\
       .ai/automation/eval

# Plugin-managed Claude subagents
rm -f .claude/agents/*.md

# Old Copilot agents (plugin generates new names)
rm -f .github/agents/AEM-After.agent.md \\
      .github/agents/AEM-Before.agent.md \\
      .github/agents/AEM-Demo.agent.md

# Convention rules -- reinstalled by init
rm -rf .claude/rules/`}</CommandBlock>
</Section>

<SectionHeading
  badge="Step 2"
  badgeColor="warning"
  title="Prep copilot-instructions.md"
  subtitle="Remove the old AEM Agents section before running aem-init."
  bg="primary"
/>
<Section>
  <ContentCard icon="mdi:text-box-remove" iconColor="bg-amber-500" title="Clean Up">
    Open <code>.github/copilot-instructions.md</code> and delete the <code>## AEM Agents</code> table block. Keep everything else. The <code>/aem-init</code> skill will append an updated section with the new agent names (<code>@AEMAfter</code> instead of <code>@AEM-After</code>).
  </ContentCard>
</Section>

<SectionHeading
  badge="Step 3"
  badgeColor="warning"
  title="Install Plugins"
  subtitle="Add the marketplace and install from it."
  bg="alt"
/>
<Section>
  <CommandBlock label="Install plugins">{`# Add the shared marketplace (one-time)
/plugin marketplace add /path/to/dx-aem-flow

# Install plugins
/plugin install dx-core@dx-aem-flow
/plugin install dx-aem@dx-aem-flow
/plugin install dx-automation@dx-aem-flow`}</CommandBlock>
</Section>

<SectionHeading
  badge="Step 4"
  badgeColor="warning"
  title="Run Init Skills"
  subtitle="Three init skills configure everything."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-3 gap-4">
    <ContentCard icon="mdi:puzzle" iconColor="bg-brand-700" title="/dx-init">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li>Auto-detects AEM Maven project</li>
        <li>Generates <code>.ai/config.yaml</code></li>
        <li>When asked about Copilot: Yes</li>
        <li>When asked about copilot-instructions: Keep existing</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:database" iconColor="bg-cyan" title="/aem-init">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li>Detects component paths and brands</li>
        <li>Writes 8 AEM rules to <code>.claude/rules/</code></li>
        <li>Writes 8 instruction files</li>
        <li>Writes 6 AEM Copilot agents</li>
        <li>Appends updated AEM Agents section</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:cog" iconColor="bg-amber-500" title="/auto-init">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">

        <li><code>infra.json</code> and <code>repos.json</code> preserved</li>
        <li>System prompt wrappers land at <code>prompts/</code></li>
        <li>Do NOT run <code>/auto-provision</code> -- infra already exists</li>
      </ul>
    </ContentCard>
  </div>
</Section>
