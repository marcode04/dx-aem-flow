/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:folder" iconColor="bg-brand-700" title="Project Structure">
      <ul class="mt-2 list-none p-0 text-sm space-y-1">
        <li><code>.ai/config.yaml</code> -- project configuration</li>
        <li><code>.ai/rules/</code> -- domain-specific override rules</li>
        <li><code>.ai/templates/</code> -- spec and ADO comment templates</li>
        <li><code>.ai/lib/</code> -- shared shell scripts</li>
        <li><code>.claude/rules/</code> -- convention rules (dual frontmatter)</li>
        <li><code>.github/agents/</code> -- 25 Copilot agent templates</li>
        <li><code>AGENTS.md</code> -- agent inventory</li>
        <li><code>.mcp.json</code> -- MCP server configuration</li>
      </ul>
    </ContentCard>
    <div>
      <HighlightBox severity="info" title="TODO Placeholders">
        Values that cannot be auto-detected (ADO/Jira project, repository GUID, build commands)
        are filled with <code>TODO</code> placeholders. Edit <code>.ai/config.yaml</code> after
        scaffolding to replace these with real values.
      </HighlightBox>

      <HighlightBox severity="warning" title="AEM Seed Data">
        With <code>--aem</code>, the scaffold also creates <code>.ai/project/</code> with empty
        seed data files. Run <code>/aem-init</code> later (when an AI tool is available) to
        populate them from the live AEM instance.
      </HighlightBox>
    </div>
  </div>
</Section>

<SectionHeading
  badge="Relationship"
  badgeColor="info"
  title="Relationship to Init Skills"
  subtitle="The scaffold output is identical to what /dx-init + /aem-init produce."
  bg="primary"
/>
<Section>
  <ContentCard icon="mdi:swap-horizontal" iconColor="bg-cyan-dark" title="Same Output, Different Mechanism" horizontal>
    The scaffold utility creates the exact same file structure as <code>/dx-init</code> and <code>/aem-init</code>. The difference is the mechanism: the scaffold runs as a plain Node.js script without any AI runtime, while the init skills run inside Claude Code or Copilot CLI and can auto-detect project values.

    You can run init skills later to re-detect and update values that the scaffold left as TODO placeholders. The init skills will not overwrite files that already have real values.
  </ContentCard>
</Section>
