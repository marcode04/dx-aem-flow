<SectionHeading
  badge="YAML"
  badgeColor="warning"
  title="project.yaml"
  subtitle="Structured project configuration -- repos, brands, markets, platforms."
  bg="alt"
/>
<Section>
  <CommandBlock label="Minimal example (single repo, single brand)">{`ado:
  org: myorg.visualstudio.com

brands:
  - name: MySite
    markets:
      - code: US
        platform: [DXN]
        aem-paths: ["/content/mysite/us/en"]
        languages: [en]
        default-language: en
        commerce: none

repos:
  - name: MySite-AEM
    platform: DXN
    ado-project: My Project
    base-branch: develop

platforms:
  - id: DXN
    resource-type-prefix: mysite/components
    css-prefix: mysite-
    component-prefixes: [mysite-]

defaults:
  qa-author-url: https://qa-author.mysite.com`}</CommandBlock>

  <HighlightBox severity="info" title="Rules">
    <ul class="text-sm list-none p-0 space-y-1">
      <li><code>ado.org</code> -- domain only, no <code>https://</code> prefix</li>
      <li><code>brands[].markets[].code</code> -- 2-letter country codes, used as keys everywhere</li>
      <li><code>brands[].markets[].aem-paths</code> -- must match actual AEM content tree</li>
      <li><code>platforms[].id</code> -- referenced by repos and markets, must match exactly</li>
    </ul>
  </HighlightBox>
</Section>

<SectionHeading
  badge="YAML"
  badgeColor="info"
  title="file-patterns.yaml"
  subtitle="Tells agents how to find source files for a component."
  bg="primary"
/>
<Section>
  <CommandBlock label="Example: DXN platform file patterns">{`DXN:
  backend:
    repo: My-Platform-Repo
    resolve-from: component-index-project
    files:
      - label: HTL template
        path: "{basePath}/{name}.html"
      - label: Dialog
        path: "{basePath}/_cq_dialog/.content.xml"
      - label: Component def
        path: "{basePath}/.content.xml"
    java-search:
      module: "my-core-module"
      patterns: ["{Name}Model.java", "{Name}ModelImpl.java"]
  frontend:
    - repo: My-Platform-Repo
      base-path: "my-clientlibs/frontend/publish/components/{name}/"
      extensions: [.clientlibs.js, .config.js, .clientlibs.scss]
      flat: true`}</CommandBlock>

  <div class="grid md:grid-cols-2 gap-4 mt-6">
    <ContentCard icon="mdi:code-braces" iconColor="bg-cyan" title="Placeholders">
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><code>{'{name}'}</code> -- lowercase component name as-is</li>
        <li><code>{'{Name}'}</code> -- PascalCase (derived from name)</li>
        <li><code>{'{basePath}'}</code> -- extracted from component-index Source Link</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:alert-circle" iconColor="bg-amber-500" title="Fallback">
      If <code>file-patterns.yaml</code> does not exist, agents fall back to generic AEM Glob patterns like <code>&#42;&#42;/components/&#42;&#42;/{'{name}'}/</code>.
    </ContentCard>
  </div>
</Section>

<SectionHeading
  badge="YAML"
  badgeColor="secondary"
  title="content-paths.yaml"
  subtitle="Tells agents how to find AEM pages that use a component."
  bg="alt"
/>
<Section>
  <CommandBlock label="Example content-paths.yaml">{`content-roots:
  pages: "/content/{brand}/{country}/{language}"
  experience-fragments: "/content/experience-fragments/{brand}/{country}/{language}"

author-url-pattern: "{qa-author-url}/editor.html{pagePath}.html"

search:
  tool: mcp__AEM__searchContent
  limit-per-path: 5
  always-search-xf: true
  primary-language-only: true

language-defaults:
  US: en
  CA: en
  MX: es
  DE: de`}</CommandBlock>

  <HighlightBox severity="info" title="Placeholders">
    <code>{'{brand}'}</code>, <code>{'{country}'}</code>, and <code>{'{language}'}</code> are substituted from <code>project.yaml</code> brand/market definitions. <code>{'{qa-author-url}'}</code> comes from <code>defaults.qa-author-url</code>.
  </HighlightBox>
</Section>

<SectionHeading
  badge="Markdown"
  badgeColor="warning"
  title="architecture.md and features.md"
  subtitle="Free-form Markdown that agents search by keyword."
  bg="primary"
/>
<Section>
  <div class="grid md:grid-cols-2 gap-4">
    <ContentCard icon="mdi:sitemap" iconColor="bg-brand-700" title="architecture.md">
      <p class="text-sm">Suggested sections:</p>
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><strong>Rendering Pipeline</strong> -- how components render (HTL, JS, hybrid)</li>
        <li><strong>Module Structure</strong> -- directory tree, Maven modules, FE build</li>
        <li><strong>Key Libraries</strong> -- shared libs, base classes, utilities</li>
        <li><strong>Key Patterns</strong> -- component anatomy, data flow, config patterns</li>
        <li><strong>Platform Differences</strong> -- Legacy vs DXN, if applicable</li>
      </ul>
    </ContentCard>
    <ContentCard icon="mdi:puzzle" iconColor="bg-cyan" title="features.md">
      <p class="text-sm">One <code>##</code> section per feature. Include:</p>
      <ul class="mt-2 text-sm list-none p-0 space-y-1">
        <li><strong>Component names</strong> -- agents cross-reference with component index</li>
        <li><strong>Config details</strong> -- CAC, OSGi, environment variables</li>
        <li><strong>Page paths</strong> -- where the feature is authored in AEM</li>
        <li><strong>Practical focus</strong> -- what a developer needs to modify the feature</li>
      </ul>
    </ContentCard>
  </div>

  <HighlightBox severity="warning" title="Rules for Free-Form Files">
    Be specific -- include actual class names, file paths, config keys. Agents grep for these. Section headings (<code>##</code>) help agents return matching sections with context. Do not duplicate structured data -- repo lists and file paths belong in the YAML files.
  </HighlightBox>
</Section>
