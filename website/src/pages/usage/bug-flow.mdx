      as features, creates PR.
    </ContentCard>

<Section>
  <div class="grid grid-cols-1 md:grid-cols-2 gap-6">
    <ContentCard icon="mdi:web" iconColor="bg-brand-700" title="Page Navigation">
      Navigates to the affected page, handles QA Basic Auth, waits for page load, and interacts
      with elements.

      - Navigate to URL
      - Click elements, fill forms
      - Take screenshots at each step
      - Evaluate JavaScript in page context
    </ContentCard>

    <ContentCard icon="mdi:database" iconColor="bg-amber-500" title="AEM Component Dialog">
      For AEM bugs, the <code>aem-bug-executor</code> agent interacts directly with the component
      dialog in the AEM author environment.

      - Open component dialog
      - Inspect field values
      - Compare with expected state
      - Screenshot dialog panels
    </ContentCard>
  </div>
</Section>

<SectionHeading
  badge="Cross-Repo"
  title="Cross-Repo Detection"
  subtitle="Bugs don't always live in the repo where they're reported."
  bg="alt"
/>
<Section>
  <HighlightBox severity="warning" title="Automatic Delegation">
    If triage discovers the fix belongs in another repository (e.g., a frontend bug caused by a backend
    issue in Brand-Backend), it writes a <code>delegate.json</code> file. The automation pipeline reads
    this and routes the bug fix to the correct repo's pipeline.
  </HighlightBox>

  <div class="grid grid-cols-1 md:grid-cols-2 gap-6 mt-4">
    <ContentCard icon="mdi:monitor" iconColor="bg-brand-700" title="Same Repo">
      Bug and fix are in the same repo. Proceed directly to verify and fix phases.
    </ContentCard>

    <ContentCard icon="mdi:swap-horizontal" iconColor="bg-amber-500" title="Other Repo">
      Bug reported in frontend but fix needed in backend. Agent writes <code>delegate.json</code>
      and the pipeline routes to the correct repo.
    </ContentCard>
  </div>
</Section>

<SectionHeading
  badge="Learning"
  badgeColor="success"
  title="Bug Pattern Detection"
  subtitle="Bug history is tracked per component to identify hotspots."
  bg="primary"
/>
<Section>
  <HighlightBox severity="error" title="Hotspot Alerts">
    When 3 or more bugs are filed against the same component, a hotspot alert is triggered. This flags the
    component for deeper review -- recurring bugs in the same area often indicate a systemic issue rather
    than isolated defects.
  </HighlightBox>
</Section>
