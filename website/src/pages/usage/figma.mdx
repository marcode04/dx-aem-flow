{/* Phase 1: Extract */}
<SectionHeading
  badge="Phase 1"
  badgeColor="info"
  title="Figma Extract"
  subtitle="/dx-figma-extract -- Pulls everything from Figma so it never needs to be consulted again."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-3 gap-4">
    <ContentCard icon="mdi:magnify" iconColor="bg-brand-700" title="Smart Node Selection">
      Auto-matches the story requirements against the Figma frame structure. Detects multi-viewport
      designs (desktop + mobile) and extracts each separately.
    </ContentCard>
    <ContentCard icon="mdi:wrench" iconColor="bg-amber-500" title="Design Quality Analysis">
      Classifies each Figma node as <em>designed</em>, <em>screenshot-context</em>, or <em>decoration</em>.
      Adapts the extraction strategy based on what it finds.
    </ContentCard>
    <ContentCard icon="mdi:palette" iconColor="bg-emerald-600" title="Token Extraction">
      Captures all design variables (colors, spacing, typography), reference screenshots, and image assets.
      Downloads everything locally for offline use.
    </ContentCard>
  </div>

  <HighlightBox severity="info" title="Output: figma-extract.md">
    Cleaned reference code (screenshot layers removed), design tokens, dynamic content annotations
    (values marked with approximate for content-dependent sizing), asset manifest, and Figma reference screenshots.
    This file is the single source of truth for all downstream steps.
  </HighlightBox>

  <div class="grid md:grid-cols-3 gap-4 mt-4">
    <ContentCard icon="mdi:star" iconColor="bg-emerald-600" title="Structured Figma">

      exceptional accuracy, near-complete token coverage.
    </ContentCard>
    <ContentCard icon="mdi:star-half-full" iconColor="bg-brand-700" title="Hybrid Figma">
      Mix of structured elements and pasted screenshots for context. The pipeline separates real design
      from reference material -- strong results with minor gaps.
    </ContentCard>
    <ContentCard icon="mdi:star-outline" iconColor="bg-amber-500" title="Screenshot-Based Figma">
      Primarily pasted screenshots with minimal structure. The pipeline reconstructs elements from images --
      solid starting point that saves significant manual effort.
    </ContentCard>
  </div>
</Section>

    >

    </ContentCard>
    <ContentCard
      icon="mdi:view-dashboard"
      iconColor="bg-cyan-dark"
      title="Template Scaffold"
      tags={[{ label: 'Consistent', color: 'info' }, { label: 'Idempotent', color: 'success' }]}
    >
      An HTML template provides the comparison layout -- side-by-side for mobile, stacked for desktop.
      The skill fills two placeholders: component HTML and Figma reference images.
    </ContentCard>
    <ContentCard
      icon="mdi:swap-horizontal"
      iconColor="bg-amber-500"
      title="Token Mapping"
      tags={[{ label: 'Traceable', color: 'warning' }, { label: 'Documented', color: 'primary' }]}
    >
      Every Figma design token is mapped to the closest project equivalent. Exact matches, close matches
      (within 10%), and unmapped values are all documented in the CSS output.
    </ContentCard>
  </div>

  <HighlightBox severity="success" title="Output">
    <code>figma-conventions.md</code> with discovered project patterns, plus <code>prototype/index.html</code>,
    <code>styles.css</code>, and <code>script.js</code>. The prototype uses the project's actual design tokens --
    not generic CSS. Open it in any browser, no build system required.
  </HighlightBox>
</Section>

{/* Phase 3: Verify */}
<SectionHeading
  badge="Phase 3"
  badgeColor="success"
  title="Figma Verify"
  subtitle="/dx-figma-verify -- Opens the prototype in a real browser, compares against the Figma reference, and auto-fixes discrepancies."
  bg="alt"
/>
<Section>
  <div class="grid md:grid-cols-3 gap-4">
    <ContentCard icon="mdi:web" iconColor="bg-brand-700" title="Browser Rendering">
      Opens the prototype in Chrome via Chrome DevTools MCP. Hides the Figma reference column, takes a
      clean screenshot of just the component output.
    </ContentCard>
    <ContentCard icon="mdi:camera" iconColor="bg-cyan-dark" title="Visual Comparison">
      Compares the prototype screenshot against <code>figma-reference.png</code> side by side. Identifies
      layout shifts, color mismatches, spacing gaps, and missing elements.
    </ContentCard>
    <ContentCard icon="mdi:hammer-wrench" iconColor="bg-amber-500" title="Auto-Fix Loop">
      If differences are found, the skill patches CSS/HTML and re-screenshots -- up to 2 iterations.
      Each round narrows the gap without manual intervention.
    </ContentCard>
  </div>

  <HighlightBox severity="success" title="Output: figma-gaps.md">
    An acceptance checklist with binary pass/fail assertions -- concrete values like "title font-size 26px",
    "button background #000", "padding-top 40px". QA can verify the component without Figma access.
    Verdict: <strong>PASS</strong>, <strong>PASS WITH MINOR GAPS</strong>, or <strong>NEEDS ATTENTION</strong>.
  </HighlightBox>
</Section>
