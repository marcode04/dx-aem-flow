<SectionHeading
  badge="Maintenance"
  badgeColor="warning"
  title="Keeping Discovery Fresh"
  subtitle="component-discovery.md captures dialog state from QA at a point in time."
  bg="primary"
/>
<Section>
  <HighlightBox severity="warning" title="Re-run /aem-init after significant changes">
    Re-scan component discovery after: major dialog changes deployed to QA, new components added, or brand overrides created.
    Choose "Re-scan component discovery" when prompted — it updates only the discovery file without reinitializing the full project.
  </HighlightBox>

  <div class="grid grid-cols-1 md:grid-cols-3 gap-6 mt-6">
    <ContentCard icon="mdi:puzzle-edit" iconColor="bg-amber-500" title="Major dialog changes">
      New fields, removed fields, or restructured tab layouts deployed to QA require a rescan so skills have accurate field semantics.
    </ContentCard>

    <ContentCard icon="mdi:plus-box" iconColor="bg-emerald-600" title="New components added">
      Components added to the project after the initial <code>/aem-init</code> run won't appear in discovery until a rescan.
    </ContentCard>

    <ContentCard icon="mdi:palette-swatch" iconColor="bg-brand-700" title="Brand overrides created">
      Brand-specific dialog overrides change the available fields per brand — update discovery to capture these variants.
    </ContentCard>
  </div>
</Section>
